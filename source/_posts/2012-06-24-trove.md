# Trove

Environment Variables are a great way of setting up environment specific configuration. However keeping track of them for each environment can be a pain and not all your developers may know how to get them (I'm looking at you AWS).

This can be especially bad if you're an agency with lots of projects.

=== read more ===

## What?
[Trove][] is small application that stores environment variables along with a name and the environment they relate to. Postgres' hstore is used to store the actual variables and [django-tastypie][] provides the API.

The sibling project, [trove-cli][], provides a command line app to access your trove of variables through the API. It defaults to writing the variables to stdout and using the `dev` environment but both can be configured in `~/.troverc`.


[django-tastypie]: https://github.com/toastdriven/django-tastypie
[Trove]: http://github.com/ghickman/trove
[trove-cli]: http://github.com/ghickman/trove-cli
